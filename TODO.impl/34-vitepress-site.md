# 34. Vitepress site — scaffold + generator + pages

**Goal**: Generate a Vitepress site under `site/` that consumes the JSON output and lets
users browse Plane → Block → Character.

**Depends on**: 29, 30.

**Files**:
- `lib/ucode/site.rb` — namespace hub.
- `lib/ucode/site/generator.rb` — orchestrator.
- `lib/ucode/site/config_emitter.rb` — writes `.vitepress/config.ts` from `output/`.
- `lib/ucode/site/search_index.rb` — builds `output/index/search.json` (MiniSearch or
      FlexSearch payload).
- `site/` — generated Vitepress app (gitignored).
- `site/.vitepress/theme/` — custom theme (Tailwind setup).
- `site/char/[codepoint].vue` — dynamic character detail page.
- `site/block/[id].vue` — dynamic block page.
- `site/plane/[n].vue` — dynamic plane page.
- `site/index.vue` — home.

## Tasks

- [ ] **Static pages generated by Ruby** (one per plane, one per block — ~363 pages):
  - `site/plane/<n>.md` — frontmatter + body that fetches `output/planes/<n>.json` and
    renders the block grid.
  - `site/block/<id>.md` — same, fetches `output/blocks/<id>.json`, renders codepoint grid
    with lazy-loaded `glyph.svg` per cell.
- [ ] **Dynamic route** for character detail (single Vue page, fetches JSON by route param):
  - `site/char/[codepoint].vue` — `useRoute()` to get `codepoint`, fetch
    `output/blocks/<id>/<cp>/index.json` and `glyph.svg`.
- [ ] **Search**: client-side search over `output/index/search.json`. Default engine:
      MiniSearch (verify bundle size in benchmark). Index built by `search_index.rb`.
- [ ] **Config emission**: sidebar + nav generated from `output/planes/*.json` and
      `output/blocks/*.json`. Top nav: 17 planes. Sidebar: blocks within current plane.
- [ ] **Theme**: Tailwind via Vite plugin. Dark mode default. Typography plugin for
  readable property tables.
- [ ] **Generator CLI**: `ucode site init` (creates `site/` skeleton) and `ucode site
  build` (regenerates static pages + search index from current `output/`).

## Acceptance criteria

- `ucode site init && ucode site build` produces a working Vitepress dev server.
- Navigating Plane 0 → Block "ASCII" → U+0041 shows the JSON properties + the SVG glyph.
- Search for "LATIN CAPITAL LETTER A" finds U+0041.
- Lighthouse score > 90 on performance + accessibility for the block page.

## Architectural notes

- **Static for planes/blocks, dynamic for chars**: ~363 static pages + 1 dynamic route.
  Generating 160 k static HTML pages is infeasible and unnecessary.
- **JSON-driven**: every page is a thin Vue shell that fetches JSON. The Ruby side
  generates the JSON + the few static pages.
- **Search index size**: 160 k codePoints × ~30 bytes each = ~5 MB. Acceptable for a
  client-side library; verify with MiniSearch benchmark.
