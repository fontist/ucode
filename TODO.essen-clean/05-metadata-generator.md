# TODO 05 — Metadata generator (`ucode emit-metadata`)

## Status

Pending. Depends on TODOs 01-04.

## Context

The per-version metadata Ruby modules (`metadata/v17_0_0.rb` etc.) are
**generated** from authoritative UCD data and committed to the gem.
This TODO covers the generator that produces them.

## Trigger

Two entry points:

1. **Standalone command**: `ucode emit-metadata --version 17.0.0`
   - Reads the cached UCD for the given version
   - Writes `lib/ucode/unicode/metadata/v17_0_0.rb`
   - Idempotent: re-running produces byte-identical output

2. **Part of parse**: `ucode parse --version 17.0.0`
   - After writing the output tree, also emits the metadata module
   - Same logic, chained after the existing parse pipeline

## Data sources (all already parsed by Coordinator)

| Field               | Source                                  |
|---------------------|-----------------------------------------|
| `UNICODE_VERSION`   | `Config::KNOWN_VERSIONS` / CLI input    |
| `ASSIGNED_COUNT`    | `DerivedGeneralCategory.txt` — count of codepoints with GC ≠ Cn, Co, Cs |
| `ASSIGNED_BY_PLANE` | Same count, grouped by `cp >> 16`       |
| `PLANES`            | `PropertyValueAliases.txt` (sc; plane)  |
| `BLOCKS`            | `Blocks.txt`                            |

## ASSIGNED_COUNT definition

Per issue #63:
> assigned = codepoints with GC ∈ {Cc, Cf, L*, M*, N*, P*, S*, Z*}
> explicitly EXCLUDES: Cn (unassigned), Co (Private_Use), Cs (Surrogate)

Source: `DerivedGeneralCategory.txt` lists only assigned codepoints.
Subtract those with `Co` or `Cs` category.

```ruby
assigned = DGC.each_record.reject { |r| ["Co", "Cs"].include?(r.gc) }
              .sum { |r| r.last_cp - r.first_cp + 1 }
```

## Generated module shape

```ruby
# frozen_string_literal: true
# AUTO-GENERATED by `ucode emit-metadata --version 17.0.0`.
# Do not edit by hand. Regenerate via: bin/ucode emit-metadata --version 17.0.0

module Ucode
  module Unicode
    module Metadata
      module V17_0_0
        UNICODE_VERSION = "17.0.0"
        ASSIGNED_COUNT = 159_866
        ASSIGNED_BY_PLANE = { 0 => 62_063, 1 => 9_027, ... }.freeze

        PLANES = [
          { number: 0,  short_name: :BMP,   display_name: "Basic Multilingual Plane", ... },
          # ... 17 entries
        ].freeze

        BLOCKS = [
          { id: "Basic_Latin", name: "Basic Latin", first_cp: 0,    last_cp: 0x7F,  plane_number: 0 },
          # ... ~346 entries
        ].freeze
      end
    end
  end
end
```

## Version → module name mapping

Dots → underscores, prefixed with `V`:

| Version    | Module name  |
|------------|--------------|
| `15.0.0`   | `V15_0_0`    |
| `15.1.0`   | `V15_1_0`    |
| `16.0.0`   | `V16_0_0`    |
| `17.0.0`   | `V17_0_0`    |

## Idempotency

The generator writes to a `.tmp` file, compares with the existing
module byte-for-byte, and only renames if content differs. Same pattern
as `Repo::AtomicWrites`.

## Files

- `lib/ucode/commands/emit_metadata.rb` (NEW) — command action class

## Acceptance criteria

- `ucode emit-metadata --version 17.0.0` writes `v17_0_0.rb`
- Re-running produces zero bytes changed (idempotent)
- `ASSIGNED_COUNT` matches the count from `DerivedGeneralCategory.txt` minus Co/Cs
- All 17 planes present in `PLANES`
- All ~346 blocks present in `BLOCKS`
- Generated file passes `ruby -c` syntax check
- No `require_relative` in generated code
